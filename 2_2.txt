Important Definitions
 Print disabled user
A person who cannot effectively read print because of a visual, 
physical, perceptual, developmental, cognitive, or learning disability, 
requiring him to utilize alternative methods to access that information.
 Screen Reader
A screen reader is a software application that renders text and 
image content as speech or braille output. They attempt to convey 
what people with normal eyesight see on a display to their users 
via non - visual means, like text - to - speech, sound icons, or a braille 
device.
 Accessible Document
Document that is created or modified in such a way that can be 
read equally by all users, whether they are print - disabled or not .

Use of good practices for text - presentation  (1)
[Word Documents]
The way the text is presented is extremely important and the right 
choices make it easier for people with reading problems (such as 
dyslexia, partial vision loss, etc.) to read it.
Recommendations
 Use of plain language (not jargon)
 Use of bulleted  or ordered lists for  g rouping relative items or 
for ordering items. 
 Use of sans - serif fonts like Arial, Helvetica, Verdana, Tahoma 
και  Trebuchet MS.
 Use font of at least  12 pt.
 Use of left aligned text, instead of justified alignment 
 Use of sufficient space between lines of a paragraph (ex.  1,5 
lines) and between paragrap hs.

Use of good practices for text - presentation  (2)
[Word Documents]
 Avoid using italics and underlining. Emphasis can be made 
using bold .
 Capitalization should not be used (for example CONTENTS, 
CHAPTER 1). B lock capitals can be difficult to follow as block 
capitals remove the natural shape of words, turning them into 
blocks. 
 It is advisable not to use abbreviations and acronyms (for 
example EE, it should be written European Union). 
Corrections in bibliographic references are excluded .
 Perform spell  check
 Remove double  spaces
 Do not use columns in your format.

Use of sufficient color contrast between text and 
background color
[Word Documents]
People with visual impairments, such as color blindness, have 
difficulty distinguishing information when there is low color contrast.
Recommendations
 Use color combinations with sufficient color contrast.
 The simple choice of black font on a white background is more than 
satisfactory.
 Use color contrast evaluation tools such as:
 Wave contrast checker
 TPGI contrast checker

Use of build - in headings  ( 1 )
[Word Documents]
 People who use screen readers, as well as people with certain 
mobility problems navigate a document using the keyboard and specific 
keys. 
 If the text is not structured properly, it becomes difficult to read.
Recommendations
 Organize headings in logical order.
 Organize document information into small sections. Ideally, each 
heading should contain only a few paragraphs.
 Use headings only to mark the structure and not to decorate the 
content.
 Paragraphs should not be styled as headings. Paragraphs should be 
‘normal’ style.

Use of build - in headings  (2)
[Word Documents]
 Embedded headings are essentially the chapters and sub - chapters of a text.
 The title of each document is marked as title.
 The document structure that can be applied is as follows:
 The title of each chapter is marked as heading 1
 As the level of detail increases, for example subheadings, so does the grade 
of the heading. i.e. heading 2, heading 3 and so on.

Add alternative text (alt text) to all graphic 
elements (1)
[Word Documents]
 People with vision problems, such as partial or total vision loss, cannot 
perceive the content of an image or some other graphic element, and as a 
result they may miss a significant part of the information provided .
 Alt text is descriptive text that conveys the meaning and context of a 
graphic in a digital environment, such as an app or web page. When screen 
readers reach digital content with alt text, they will read the alt text aloud, 
allowing users to better understand what's on the screen. Alt text greatly 
reduces ambiguity and improves the user experience.
Recommendations
 Add alt text to all graphic elements (pictures, SmartArt graphics, shapes, 
groups, charts, embedded objects). 
 The alt text is a short description of the content of the image. It's usually 1 -
2 lines. 
 If the image has only a decorative role, it is marked accordingly and no 
alternative text is required.

Text in pictures is not used as the only mean 
of conveying important information
[Word Documents]
 Text in an image cannot be read by screen readers. Therefore, if there 
is important information in the image, it is inaccessible.
Recommendations
If there is important information with text in the image, you should:
 Repeat this text in the document.
 In the alternative text of the image, a brief description of the image 
and the existence of the supplementary text and its purpose should be 
mentioned.

Using  Colour
[ Word Documents]
People with visual impairments, such as blindness, color blindness or low 
vision, may not be able to perceive information provided only using color. 
As a result c olor should not be used as the only visual means of 
conveying information.
Recommendations
When there is a hyperlink, underlining should be used, so that people with 
visual impairments, such as color blindness, realize that the text is a 
hyperlink.

Use of meaningful hyperlink text and 
ScreenTips(1)
[ Word Documents]
People who use screen readers often check the text to find the list of 
hyperlinks it has. If the hyperlink text is not indicative of the 
destination, it creates confusion. For example:
 For more information click on the link  here.
 For more information click on the  Official Social Welfare Website
link.
Recommendations
 Hyperlinks should convey clear and accurate information about 
their destination.
 The hyperlink should be checked for the validity of its destination. 
 Add a screen tip, equivalent to the alt text for the images, to 
describe what the  web page you are linking to is .

Use of simple structure for the tables and

Tables should be well structured

Use of build - in accessibility checker  (1)
[ Word Documents]
The Accessibility Checker verifies your file against a set of rules that identify 
possible issues for people who have disabilities. Depending on how severe 
the issue is, the Accessibility Checker classifies each issue as an error, 
warning, or tip .
 Error . Content that makes the document difficult or impossible to read 
and understand for people with disabilities
 Warning . Content that in most (but not all) cases makes the document 
difficult to understand for people with disabilities
 Tip . Content that people with disabilities can understand but that could 
be presented in a different way to improve the user’s experience
 Intelligent Services. Content that is automatically made accessible by 
AI, and that you should review for accuracy and context

Web Accessibility Directive
 The Web Accessibility Directive (Directive (EU) 2016/2102) has been in 
force since 22 December 2016.
 The Directive obliges websites and apps of public sector bodies to meet 
specific technical accessibility standards. There are a limited number of 
exceptions that include broadcasters and live streaming.
The Directive requires
 An accessibility statement for each website and mobile  app.  Public 
organizations websites must comply with accessibility guidelines
 A  feedback mechanism so users can flag accessibility problems or 
request information published in a non - accessible content.
 Regular monitoring of public sector websites and apps by Member States 
and reporting on the results.

Steps for Web Accessibility
1. Design a Website or Web App with Accessibility in mind.
2. Be sure that the criteria of Web Accessibility are met 
throughout the development process.
3. Evaluate accessibility using different types of tests
1. Automated tools
2. Evaluation by the designer / developer
3. Human evaluation by real users
4. Evaluation by external partners
4. Upload only accessible documents (docx, pdf, excel files)
5. Be sure that the content that is being upload on the Website/ 
Web App also meets the criteria of accessibility.

Four Principles of Accessibility (1)
 Perceivable
Information and user interface components must be presentable to 
users in ways they can perceive
Examples
 Provide alt text on images
 Provide subtitles to videos
 Use proper color contrast between background and  font
 Operable
User interface components and navigation must be operable.  The 
elements that make up the user interface and navigation elements 
must be functional. The interface should not require interaction that 
the user cannot perform.
Examples
 All functionality of the content is operable through a keyboard interface
 There should be no content presented in a way that is known to cause 
seizures or physical reactions
 Make it easy for the user to find the content they are looking for and 
navigate the site.

Four Principles of Accessibility (2)
 Understandable
Information and the operation of the user interface must be 
understandable
Examples
 Content to be displayed in a predictable manner
 Help users when filling out forms or when correcting errors that 
may occur
 Robust
Content must be robust enough that it can be interpreted 
reliably by a wide variety of user agents, including assistive 
technologies
Examples
 Application of techniques that allow the use of the website from 
different devices (such as smart phones, tablets, etc.)
 Application of techniques that allow the use of the website by 
assistive technology for People with Disabilities.
